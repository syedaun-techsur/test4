**Task data shape (from TechArch §3.3):**
```typescript
interface Task {
  id: string;          // crypto.randomUUID() or Date.now().toString(36) + Math.random().toString(36).slice(2)
  text: string;        // 1-500 chars, trimmed
  completed: boolean;  // always false on creation
  createdAt: string;   // ISO 8601, new Date().toISOString()
}
```

**Error classes (from FRD §Y1):**
```javascript
class ValidationError extends Error {
  constructor(code, message) {
    super(message);
    this.name = "ValidationError";
    this.code = code; // "EMPTY_TASK_TEXT" | "TASK_TEXT_TOO_LONG"
  }
}
class TaskNotFoundError extends Error {
  constructor(taskId) {
    super(`Task not found: ${taskId}`);
    this.name = "TaskNotFoundError";
    this.code = "TASK_NOT_FOUND";
  }
}
class StorageError extends Error {
  constructor(message) {
    super(message);
    this.name = "StorageError";
    this.code = "STORAGE_WRITE_FAILED";
  }
}
```

**Store functions (from TechArch §4.2):**
```typescript
hydrateFromStorage(): void  // Phase 1: reads nothing, sets taskList = []
getTasks(): Task[]           // returns shallow copy of taskList (newest first)
addTask(text: string): Task  // validates, creates, prepends, returns Task; Phase 1: no localStorage
toggleTaskComplete(id: string): Task  // flips completed, returns updated Task; Phase 1: no localStorage
deleteTask(id: string): void          // removes task; Phase 1: no localStorage
```

**Validation rules for addTask (from FRD §F0):**
- `text.trim() === ""` → throw `ValidationError("EMPTY_TASK_TEXT", "Task description cannot be empty.")`
- `text.trim().length > 500` → throw `ValidationError("TASK_TEXT_TOO_LONG", "Task description must be 500 characters or fewer.")`

**ID generation (from TechArch §4.5):**
```javascript
function generateId() {
  if (typeof crypto !== "undefined" && typeof crypto.randomUUID === "function") {
    return crypto.randomUUID();
  }
  return Date.now().toString(36) + Math.random().toString(36).slice(2);
}
// Collision guard: check against existing taskList before use
```
