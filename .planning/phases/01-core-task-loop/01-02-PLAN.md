<task type="auto">
  <name>Task 1: Create index.html app shell with all required element IDs</name>
  <files>todoapp/index.html</files>
  <action>
Create `todoapp/index.html`. The file structure must be exactly as specified in TechArch §2.1 and §2.2.

**Required DOM elements (from TechArch §2.2 — copy EXACTLY):**
- `id="task-input"` — `<input type="text">` for new task description; include `maxlength="500"` (FRD §F0) and `autofocus` attribute (US-0.2); placeholder text: "Add a new task..."
- `id="add-btn"` — submit button, text: "Add"
- `id="error-msg"` — inline validation error container; initially hidden via CSS class; `role="alert"` for accessibility
- `id="storage-notice"` — storage warning banner; initially hidden via CSS class; `role="status"` for accessibility
- `id="task-list"` — `<ul>` element repopulated on every render

**Full HTML structure:**
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="Content-Security-Policy"
        content="default-src 'self'; script-src 'self'; style-src 'self'; object-src 'none';">
  <title>TodoApp</title>
  <link rel="stylesheet" href="css/styles.css">
</head>
<body>
  <div id="storage-notice" class="storage-notice hidden" role="status" aria-live="polite"></div>

  <main class="app-container">
    <h1>My Tasks</h1>

    <section class="add-task-section" aria-label="Add new task">
      <div class="input-row">
        <input
          type="text"
          id="task-input"
          name="task-input"
          placeholder="Add a new task..."
          maxlength="500"
          autofocus
          autocomplete="off"
          aria-label="New task description"
          aria-describedby="error-msg"
        >
        <button type="button" id="add-btn">Add</button>
      </div>
      <p id="error-msg" class="error-msg hidden" role="alert" aria-live="assertive"></p>
    </section>

    <section class="task-list-section" aria-label="Task list">
      <ul id="task-list" aria-live="polite" aria-label="Tasks"></ul>
    </section>
  </main>

  <script type="module" src="js/main.js"></script>
</body>
</html>
```

**Notes:**
- `<script type="module">` is required — taskStore.js uses ES module exports
- The CSP meta tag is from TechArch §5.5
- `aria-live="polite"` on #task-list for screen reader updates
- `aria-describedby="error-msg"` on input for accessibility
- Do NOT inline any JavaScript or styles — all JS is in js/main.js, all styles in css/styles.css

<task type="auto">
  <name>Task 2: Create styles.css with full visual design including completion, error, and empty state styles</name>
  <files>todoapp/css/styles.css</files>
  <action>
Create `todoapp/css/styles.css`. Must include all styles needed for the full UI — the ui.js renderer will rely on these CSS classes for completion state, error display, and empty state.

**Required CSS classes and their purpose:**

1. **Layout:**
   - `body`: centered content, clean background (`#f5f5f5` or similar), system font stack
   - `.app-container`: max-width ~600px, centered with `margin: 0 auto`, padding

2. **Add task section:**
   - `.input-row`: flex row, gap between input and button
   - `#task-input`: flex-grow 1, padding, border, border-radius, font-size match body
   - `#add-btn`: padding, background color, white text, border-radius, cursor pointer; hover state darkens background
   - Input focus style: visible outline (accessibility)

3. **Error and notice (from FRD §Y2 + US-0.3, US-0.4):**
   - `.error-msg`: red text, small font, margin-top for spacing below input
   - `.storage-notice`: amber/yellow background banner, padding, full-width at top of page
   - `.hidden`: `display: none` — used to hide #error-msg and #storage-notice initially

4. **Task list items (will be created dynamically by ui.js):**
   - `#task-list`: list-style none, padding 0, margin-top for spacing
   - `.task-item`: flex row, align-items center, gap, padding, background white, border-radius, margin-bottom, subtle border
   - `.task-toggle` (checkbox): cursor pointer, flex-shrink 0, width/height for easy click target
   - `.task-text`: flex-grow 1, font-size, cursor default
   - `.task-delete`: button, cursor pointer, background transparent, border none, color muted gray, font-size, padding; hover color darkens to red/danger

5. **Completion state (from FRD §F1 + US-1.2, F2):**
   - `.task-item.completed .task-text`: `text-decoration: line-through` + `color: #999` (muted gray)
   - `.task-item.completed`: optional subtle opacity reduction

6. **Empty state (from FRD §F1, US-1.3 — EXACT text must be "No tasks yet. Add one above!"):**
   - `.empty-state`: text-align center, color muted, font-style italic, padding for breathing room
   - (The text content itself is set in ui.js; this class just styles the container)

7. **Responsive:**
   - At small viewports (≤480px): `.input-row` wraps or stacks; input full width

**XSS note from TechArch §5.3:** Task text is always set via `.textContent` (in ui.js) — no innerHTML — so no need for CSS escaping.

**CSS variable (optional but clean):**
```css
:root {
  --color-primary: #4a90d9;
  --color-danger: #e74c3c;
  --color-muted: #999;
  --color-completed-text: #999;
  --color-border: #ddd;
  --color-bg: #f5f5f5;
  --color-card-bg: #fff;
}
```

**Minimum content:** at least 50 lines of actual CSS rules (not counting blank lines or comments). The file should be self-sufficient — opening the page without JS should still show a well-styled form.
